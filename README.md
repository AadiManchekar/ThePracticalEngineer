## Overview

**The Practical Engineer** is a curated collection of real-world engineering knowledge derived from production systems, debugging experiences, and deep dives into distributed systems.

---

## Purpose

This repository exists to:

- Capture real production issues and their fixes
- Build strong debugging intuition
- Document system design trade-offs
- Break down complex engineering concepts into usable insights
- Serve as a long-term engineering knowledge base

---

## Repository Structure

### 01 - Production Incidents
Real-world failures and issues observed in systems.

Includes:
- Root cause analysis
- Debugging approach
- Fix implementation
- Prevention strategies

---

### 02 - Research Papers
Deep dives into:

- Databases (LSM trees, indexing, transactions)
- Message brokers (Kafka internals, delivery semantics)
- Distributed systems (CAP theorem, consensus)
- Algorithms (Bloom filters, consistent hashing)
- Research papers (Dynamo, Bigtable)

---

### 03 - Patterns and Anti-Patterns

- Proven engineering patterns (Circuit Breaker, Retry)
- Common mistakes (Shared mutable state, God services)

---

### 04 - Debugging Playbook

Practical guides for diagnosing:

- High latency
- CPU spikes
- Memory leaks
- Thread contention
- GC issues

---

### 05 - Performance Engineering

- Capacity planning
- Load testing strategies
- Throughput vs latency analysis
- Applying Little’s Law in real systems

---

### 06 - Architecture Notes

- Monolith vs Microservices
- Event-driven systems
- Scalability trade-offs
- Real-world design decisions

---

## Philosophy

- Prefer **clarity over cleverness**
- Prefer **real-world experience over theory**
- Prefer **simple systems that scale**
- Always understand **why something works**

---

## Disclaimer

These notes are based on real-world experience and research.

They represent practical approaches, not absolute truths.  
Engineering decisions are always context-dependent.
